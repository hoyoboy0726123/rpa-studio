# RPA Studio 使用 SOP(實測教學)

> 本工具的「真實錄製 / 重播 / MFA 暫停」需要你在互動桌面上實際操作(無法自動驗證),
> 以下是手把手步驟。建議照「§10 第一次完整實測」走一遍最快上手。

---

## 0. 前置(一次性)
- 需求:Windows 10/11 + 已安裝 Python 3.10+(安裝時要勾「Add Python to PATH」)。
- 啟動:到 `apps\rpa_studio\` **雙擊 `start.bat`**。
  - 第一次會自動:建虛擬環境 `venv` → 安裝依賴 → 下載 Playwright Chromium(約 2–5 分鐘)→ 開啟視窗。
  - 之後雙擊就直接開。
- 若閃退:多半是沒裝 Python 或沒在 PATH;裝好再雙擊。

## 1. 介面導覽(9 頁)
| 頁 | 做什麼 |
|----|--------|
| 流程清單 | 看所有流程、載入內建範例 |
| 編輯器 | 用表單新增/編輯流程的每個步驟 |
| 流程圖 | 同一流程的視覺節點圖,點節點編輯、連 goto、拖拉重排 |
| 錄製 | 錄製 web / 桌面操作自動生成流程 |
| 執行 | 選流程、執行、看即時日誌、停止、MFA 繼續 |
| 排程 | 用 Windows 工作排程器定時自動跑 |
| 憑證 | 安全保管帳密(Vault),流程只引用名稱 |
| 日誌 | 最近執行紀錄 |
| 執行報表 | 每次執行的逐步結果 + 自癒紀錄,可匯出 Excel |

## 2. 練習 A:錄一個「網頁登入」流程(最推薦先做)
用一個公開的自動化練習站來學(不需要你的內網)。

1. 開 RPA Studio → 左側點 **「錄製」**。
2. 引擎選 **Web**;URL 填練習站 `https://www.saucedemo.com/`(這是公開的 demo 站);流程名稱填 `web_練習`。
3. 按 **「開始錄製」** → 會另外彈出一個**瀏覽器視窗**(這是 Playwright 的錄製器)。
4. 在那個瀏覽器裡**實際操作**:帳號輸入 `standard_user`、密碼 `secret_sauce` → 按 **Login** → 點一兩個商品 → 加入購物車。
5. 操作完 → **關掉那個瀏覽器視窗**(關閉即代表錄製結束)。
6. 回到 RPA Studio,「錄製」頁會顯示**抓到的步驟預覽** → 按 **「存成流程」**。
7. 左側點 **「執行」** → 流程選 `web_練習` → 按 **「執行」**。
   - 看「即時日誌」逐步前進、進度條跑;它會自動重開瀏覽器把你的操作**重播**一遍。
8. 左側點 **「執行報表」** → 看這次每一步的狀態 / 耗時 / 有沒有失敗。

✅ 成功重播 = 你已學會「錄製 → 存 → 執行 → 看報表」整個循環。

---

## 3. 練習 B:桌面錄製(用記事本)
1. 「錄製」→ 引擎選 **Desktop** → 流程名稱 `desktop_練習` → **開始錄製**。
2. 手動打開**記事本**,點編輯區、打字、點功能表等。
   - 每次點擊它會**同時抓** UIA 元素 + 元素周圍小截圖(影像錨點)+ 座標。
   - ⚠️ 錄製時不要去點 RPA Studio 自己的視窗(它在全域監聽鍵鼠)。
3. 按 **F9** 停止(或回頁面按停止)。→ 預覽步驟 → **存成流程**。
4. 「執行」頁選 `desktop_練習` → 執行,看它自動重播。

---

## 4. 不用錄製,手動建/改流程
- **編輯器**頁:上方選(或新建)流程 → 左側步驟清單用「＋新增 / 刪除 / ↑↓ 移動」→ 右側表單編輯選定步驟:
  - 動作 action(下拉:`web.*` 網頁、`desktop.*` 桌面、`flow.*` 控制流)
  - 標籤、on_error(abort 中止 / continue 略過 / goto:步驟id)、timeout、retry、params(key-value)、target(定位器)
  - 改完按 **存檔**。
- **流程圖**頁:同一條流程的視覺版,點節點編輯、用「連 goto」拉錯誤跳轉線、拖拉節點重排、滾輪縮放、空白處拖曳平移。

---

## 5. 帳密怎麼安全帶(Vault + secret_ref)
1. **「憑證」**頁:新增一筆 secret,例如 名稱 `erp_pwd`、值 = 你的密碼 → 儲存。
   - 值存進 **Windows 憑證管理員**,不會明文落地、不會寫進流程檔。
2. **編輯器**:把「輸入密碼」那個 `web.fill`(或 `desktop.type`)步驟的 **secret_ref** 設成 `erp_pwd`(params 的 value 留空)。
   - 執行時系統自動把密碼帶入該步,畫面與日誌都不會顯示明文。

---

## 6. 需要人工/MFA 的步驟
1. **編輯器**在需要等人的位置插一個步驟,action 選 **`flow.pause_for_human`**,params 的 message 寫提示(例「請完成手機 OTP 後按繼續」)。
2. **執行**時跑到該步會**暫停**:執行頁出現黃色橫幅 +「繼續」鈕,右下角狀態燈轉**橙色 PAUSED**。
3. 你完成 MFA/驗證後按 **「繼續」**,流程接著跑。
   - (無人值守模式下這步會自動跳過,見 §9。)

---

## 7. 多層定位 & 自癒(自動,你只要會看)
- 錄製時已自動存多組定位器(桌面:UIA+影像+座標;網頁:role/text+css)。
- 回放某步定位失敗 → 自動往下一層 fallback;全失敗 → **self-healing** 用指紋在當前畫面找最像的元素替換,並記一筆到 **執行報表 → heal_logs**。
- 看到 heal 紀錄時:去確認它「猜對了沒」。猜對 → 可回編輯器把該步定位器更新成更穩的;猜錯 → 修正定位器。**系統不會自動改你的流程檔**,一切等你審核。

---

## 8. 排程(定時自動跑)
1. **「排程」**頁:選流程、頻率(每日/每週/每月)、時間 → **建立排程**。
   - 它會用 **Windows 工作排程器**建一個 `RPAStudio_<流程名>` 任務,內容 = `run_cli.py --flow <名> --unattended`。
2. ⚠️ **建立/刪除排程需系統管理員權限** → 請**以系統管理員身分**啟動 RPA Studio(對 `start.bat` 按右鍵→以系統管理員執行)。列出/檢視不用。

---

## 9. 命令列 / 無人值守(進階,可不開 UI)
在 `apps\rpa_studio\` 開命令提示字元:
```
venv\Scripts\python run_cli.py --flow 流程名稱
venv\Scripts\python run_cli.py --flow 流程名稱 --unattended
venv\Scripts\python run_cli.py --flow 流程名稱 --service-account erp_login --var month=2026-05
```
- `--unattended`:`flow.pause_for_human` **不等人、直接續跑**(前提:該系統免 MFA 或用服務帳號)。
- `--service-account <secret名>`:從 Vault 取服務帳號注入。
- `--var k=v`:覆寫流程變數(可多個)。
- 此路徑**不需要 PySide6**,可在沒裝 UI 的機器上跑(無人值守用)。

---

## 10. 第一次完整實測(建議照這個跑一遍,約 20 分鐘)
1. 雙擊 `start.bat`,等首次安裝完、視窗開起來。
2. 照 **§2** 對 `saucedemo.com` 錄一段登入 → 存成 `web_練習`。
3. 「執行」跑 `web_練習`,確認重播成功、「執行報表」看到綠色步驟。
4. 照 **§5** 在「憑證」加一筆 secret,回「編輯器」把密碼步驟改成用 secret_ref,再跑一次。
5. 照 **§6** 插一個 `flow.pause_for_human`,跑到時按「繼續」,體驗 MFA 暫停。
6. (選)照 **§3** 錄一個記事本桌面流程。
完成以上,你就會用整套工具了。

---

## 疑難排解
- **瀏覽器沒跳出來**:首次 `start.bat` 要完整跑完 `playwright install chromium`;沒裝成功就不會開瀏覽器。
- **桌面錄製抓不到元素**:UWP/系統視窗的 UIA 較弱,會自動退到影像/座標;盡量錄有標準控制項的程式(如 classic 記事本)。
- **schtasks 拒絕存取**:用系統管理員開 RPA Studio。
- **重播時定位失敗**:看「執行報表」有沒有 heal 紀錄;若該步是純座標,改用有 UIA/影像的定位較穩。
- **真內網系統**:把練習站 URL 換成你的內網網址即可;登入若有 MFA,用 §6 的人工暫停或 §9 的服務帳號。
